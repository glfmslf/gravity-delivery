# 准时送达评级设计

## 目标

在成功通关后给出直观的 `S/A/B/C` 评级，让玩家理解剩余时间、连续投递和撞击次数如何影响表现，并增强通关反馈。

失败流程不计算评级，继续沿用当前失败界面。

## 评级规则

新增纯函数 `calculateDeliveryRating()`，接收：

- `timeLeft`：通关时剩余秒数
- `maxCombo`：本局最大连击
- `hitCount`：本局撞击次数
- `totalDeliveries`：关卡订单总数

按以下优先级返回评级和中文说明：

1. `S`：`hitCount === 0` 且 `maxCombo >= totalDeliveries`，说明为“完美送达”。
2. `A`：`hitCount <= 1` 且 `timeLeft >= 30`，说明为“准时送达”。
3. `B`：`timeLeft >= 15`，说明为“顺利送达”。
4. `C`：其他成功通关情况，说明为“完成送达”。

评级基于可理解的局内表现，不直接按总分切分，避免不同关卡基础分差异影响评价。

## 状态与数据流

成功判定后，游戏同时计算最终分数和评级，并写入当前游戏状态。排行榜记录新增 `rating` 字段，与分数、关卡、连击和撞击数据一起保存到 `localStorage`。

读取旧排行榜记录时不做迁移；缺少评级的记录显示 `--`，保持向后兼容。

## 成功界面

采用“大字评级”布局：

- 顶部显示“配送成功”。
- 中央用大号字显示评级。
- 评级下方显示说明文案。
- 底部展示分数、剩余时间、最大连击和撞击次数。
- 保留“Space / Enter / 点击进入下一关”提示。

失败界面保持当前视觉和信息结构。
