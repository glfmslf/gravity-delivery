# 街机撞击反馈设计

## 目标

让撞到障碍具有明显的街机冲击感，同时保持反重力操作可读、可控，不让特效遮挡后续障碍。

保留现有规则：撞击扣除 4 秒、中断连击、玩家反弹、进入撞击冷却并播放撞击音效。

## 撞击停顿

有效撞击触发 `60ms` 停顿。停顿期间：

- 冻结倒计时、关卡距离、玩家物理和碰撞判定。
- 游戏仍读取并保留重力翻转输入，停顿结束后的首个更新帧执行。
- 背景音乐和音效不暂停。
- 结束态与重新开始流程不使用撞击停顿。

停顿使用游戏状态中的剩余时间计数，不通过阻塞主线程实现。

## 视觉反馈

有效撞击同时启动约 `320ms` 的效果生命周期：

- Canvas 产生快速衰减的强震动，初始最大位移约 12 像素。
- 全屏红色闪光在约 `180ms` 内消退，并带暗红色边缘。
- 撞击点生成 10 个黄白、橙红火花，主要向玩家后方扩散。
- 火花具有固定初速度、重力和生命周期，避免每帧随机导致画面跳变。
- 现有玩家无敌闪烁继续保留。

视觉效果通过 Canvas 状态驱动，不增加 DOM 叠层。

## 模块边界

新增 `src/hit-effects.js`，提供纯函数：

- 创建撞击反馈状态。
- 推进停顿、震动、闪光和火花时间。
- 计算当前震动偏移和闪光强度。

`src/game.js` 负责在有效撞击时创建状态并在每帧推进。`src/renderer.js` 只读取效果状态，应用场景偏移并绘制闪光和火花。

## 测试与验收

- 验证撞击创建 `60ms` 停顿和 `320ms` 效果。
- 验证停顿时间按帧递减且不会小于零。
- 验证震动和闪光随时间衰减。
- 验证火花数量、方向和生命周期稳定。
- 运行全部 Node 测试。
- 在桌面和移动视口试玩，确认撞击反馈明显且不产生横向溢出或持续遮挡。
