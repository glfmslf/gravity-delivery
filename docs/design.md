# 反重力快递设计文档

## 项目定位

《反重力快递》是一款轻量网页动作挑战小游戏。玩家驾驶一台绑着纸箱的小型快递飞行器，在城市楼道、管道和招牌之间穿梭，在倒计时结束前把包裹送到终点。

第一阶段目标是做出一个能玩、能展示、能继续迭代的 MVP，而不是一次性完成完整商业游戏。

## 目标玩家体验

玩家应该在 10 秒内理解规则，在 1 分钟内进入紧张状态，并在失败后愿意立刻重开。

核心感受：

- 切换重力带来明显但可控的方向反转反馈。
- 玩家主要判断上下通道，在合适时机翻转重力，靠近绿色门牌完成投递，而不是横向驾驶。
- 时间机制让主题成立：玩家不是单纯活下去，而是在赶单。
- 撞击扣时间让第一版更宽容，方便调试和试错。

## MVP 范围

第一版包含：

- 一个网页 Canvas 游戏页面
- 一台可控制的快递飞行器
- 空格切换重力
- 一个限时送达关卡
- 多个手工关卡
- 障碍物
- 投递门牌
- 时间包
- 撞击扣时间
- 到达终点成功
- 时间归零失败
- 成功/失败状态界面
- 基础音效反馈
- 程序化背景音乐和声音开关
- 重开按钮
- 本机排行榜
- 连击计分
- 准时送达评级
- SVG 图片资源

第一版不包含：

- 账号或存档
- 商店和成长系统
- 复杂剧情
- 移动端深度适配

## 核心规则

### 操作

- `Space`：切换重力方向
- 点击画布或“开始 / 翻转”按钮：切换重力方向
- `Enter`：开始或结束后重开

玩家横向位置固定，关卡自动向前滚动。

### 胜利

玩家在时间归零前完成全部投递门牌，并到达关卡终点，配送成功。

### 失败

倒计时归零，或到达终点时仍有漏送订单，配送失败。

### 时间变化

- 初始时间：60 秒
- 完成投递门牌：增加时间
- 拾取时间包：增加时间
- 撞击障碍：扣除时间，并短暂无敌或冷却，避免连续扣除
- 有效撞击触发 60ms 非阻塞停顿、强震动、红色闪光和固定轨迹火花；停顿期间保留重力翻转输入

### 评分

- 连续完成投递会增加当前连击，并记录最大连击。
- 撞击障碍会中断当前连击，并增加撞击次数。
- 成功通关后，排行榜分数综合关卡、完成投递数、剩余时间、最大连击和撞击惩罚。
- 成功结算显示 `S/A/B/C` 评级：全程零撞击且保持全连击为 `S`；最多一次撞击且剩余至少 30 秒为 `A`；剩余至少 15 秒为 `B`；其他成功通关为 `C`。

具体数值在开发中调参确定。

## 视觉方向

风格是“纸箱快递 + 城市楼道”：

- 主角像小型快递飞行器，绑着纸箱。
- 背景是城市楼道、楼层、管道、招牌。
- 色彩使用暖色纸箱、深色楼道、青绿色投递门牌、黄色时间包和橙红色危险元素。
- 第一版使用 SVG 图片资源绘制核心对象，并保留 Canvas 几何绘制兜底。

## 技术方案

第一阶段使用原生前端：

- `HTML` 提供页面结构
- `CSS` 提供布局和视觉样式
- `JavaScript` + `Canvas` 实现游戏循环、绘制和交互

暂时不引入 React、构建工具或游戏引擎。这样启动成本低，调试直接，适合快速验证玩法。

## 模块职责

`src/main.js` 负责连接 DOM、创建游戏实例和绑定重开按钮。

`src/game.js` 负责游戏主循环、状态流转、倒计时、胜负判断和 UI 更新。

`src/input.js` 负责键盘输入状态。

`src/player.js` 负责玩家位置、速度、重力方向和边界处理。

`src/level.js` 负责关卡长度、速度、障碍物、投递门牌和拾取物数据。

`src/renderer.js` 负责 Canvas 绘制。

`src/audio.js` 负责基础 WebAudio 音效反馈、配送中背景音乐和静音开关。

`src/assets.js` 负责图片资源清单和预加载，加载失败时由绘制层回退到几何图形。

`src/leaderboard.js` 负责本机排行榜计分、排序和 `localStorage` 持久化。

`src/rating.js` 负责根据剩余时间、最大连击和撞击次数计算准时送达评级。

`src/hit-effects.js` 负责撞击停顿、震动、闪光和火花的纯状态计算。
