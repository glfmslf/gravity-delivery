# 反重力快递开发路线图

## 里程碑 1：项目启动

目标：让项目有清晰边界和可运行入口。

- 建立项目结构
- 写 `README.md`
- 写 `docs/design.md`
- 写 `docs/roadmap.md`
- 建立 Canvas 页面
- 建立最小游戏循环

完成标准：打开 `index.html` 能看到游戏画面，玩家可以移动和切换重力。

## 里程碑 2：核心手感

目标：让移动、下坠和重力切换有基本手感。

- 调整横向移动速度
- 调整重力加速度
- 调整撞到上下边界时的反馈
- 加入更明确的重力方向视觉提示

完成标准：玩家能稳定理解并控制飞行器。

## 里程碑 3：关卡与障碍

目标：让游戏从“能动”变成“有挑战”。

- 设计第一版障碍数据结构
- 绘制障碍物
- 实现碰撞检测
- 撞击扣时间
- 加入撞击冷却

完成标准：玩家需要主动躲避障碍，撞击会产生明确代价。

## 里程碑 4：时间包与送达

目标：让限时送货主题成立。

- 添加时间包
- 实现拾取检测
- 拾取后增加倒计时
- 关卡到达终点后成功
- 时间归零后失败

完成标准：玩家可以通过拾取时间包提高送达成功率。

## 里程碑 5：第一版体验整理

目标：让 MVP 可以给别人试玩。

- 开始状态
- 成功状态
- 失败状态
- 重开流程
- 基础音效
- 视觉细节打磨
- 写试玩说明

完成标准：别人不看代码也能打开并完成一次试玩。
